# SQL_Practice_Activities

SQL Filtering and Relational Joins Activity Project Overview This repository contains a collection of SQL scripts developed for a database management activity. The project is divided into two main parts: Customer Data Filtering: Demonstrates proficiency in using WHERE clauses, logical operators (AND, OR, IN), and pattern matching (LIKE).

Hospital Management System Joins: Focuses on relational database concepts, specifically using INNER JOIN and LEFT JOIN to connect patient records with appointment schedules.

Scenarios

Customer Membership Analysis The filtering queries are based on a Customers table containing data such as membership levels (Gold, Silver, Bronze), cities (Cebu, Bohol, Tagbilaran), and join dates. Key Tasks: Filtering by location, membership status, and date ranges.
Hospital Management System This scenario involves two tables: Patients: Stores patient IDs, names, and contact details. Appointments: Stores appointment IDs, assigned doctors, and statuses (Completed, Scheduled, or Cancelled). Key Tasks: Retrieving combined records, filtering appointments by doctor, and handling missing data using analytical join queries.
Files in this Repository filtering_activity.sql: Contains the queries for the customer filtering tasks. join_operations.sql: Contains the queries for the hospital system join tasks.

How to UseThese queries are written in standard SQL and can be executed in database management environments like MySQL or PostgreSQL.
